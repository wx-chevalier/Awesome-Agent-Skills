# /competitive-analysis -- 竞争格局分析

调研并分析你的竞争格局。识别直接竞品与间接竞品，梳理定位，发现差异化机会。

## 调用方式

```
/competitive-analysis AI 驱动的项目管理工具
/competitive-analysis 我们的产品 vs Notion、Asana 和 Monday.com
/competitive-analysis [上传竞品列表或市场简报]
```

## 工作流程

### 第一步：理解竞争背景

询问用户：
- 你的产品是什么？它属于哪个品类？
- 有指定要分析的竞品吗？还是让我来识别？
- 分析的侧重点是什么？（功能对比、定位、定价、市场进入策略）
- 这份分析用于什么目的？（战略制定、销售赋能、融资路演、产品路线图）

### 第二步：识别竞品

应用 **competitor-analysis** 技能：

- 识别 5 个直接竞品（同品类、同目标买家）
- 识别 2-3 个间接竞品（不同方式、相同的待完成工作）
- 如有必要，注明新兴颠覆性玩家
- 通过网络调研获取最新信息

### 第三步：逐一分析竞品

针对每家竞品：
- **定位**：他们如何描述自己、目标受众、核心信息
- **优势**：做得好的地方、赢单的场景
- **劣势**：不足之处、常见投诉
- **定价**：定价模式及价格区间（如公开披露）
- **市场牵引力**：融资情况、团队规模、客户基础信号
- **近期动向**：新功能、合作伙伴、战略转型

### 第四步：生成竞品分析报告

```
## 竞品分析：[你的产品/市场]

**日期**：[今天]
**分析竞品数**：[数量]

### 市场概览
[2-3 句话描述市场动态、趋势及发展方向]

### 竞争格局
| 竞品 | 品类 | 目标客户 | 定位 | 优势 | 劣势 |
|------|------|----------|------|------|------|

### 功能对比矩阵
| 能力项 | 你的产品 | 竞品 A | 竞品 B | 竞品 C |
|--------|----------|--------|--------|--------|

### 定位地图
[2x2 矩阵，展示各竞品在关键维度上的定位]

### 差异化机会
1. **[机会]** — [为何具有防御性和价值]
2. ...

### 竞争威胁
1. **[威胁]** — [需关注的信号，建议应对策略]
2. ...

### 建议
- **持续强化**：[你的独特优势]
- **补齐短板**：[你缺少的基础功能]
- **选择忽略**：[不值得跟进的竞品动作]
```

保存为 markdown 格式。

### 第五步：提供后续建议

- "要我针对某个具体竞品**制作销售对战卡**吗？"
- "要我**制定差异化定位**，与头部竞品形成区隔吗？"
- "要我**识别功能缺口**，添加到路线图中吗？"

## 注意事项

- 竞品数据来源于网络调研——结果的时效性取决于可获取的信息源
- 区分"入场门槛"（参与竞争的必要条件）与"差异化因素"（赢得竞争的关键）
- 不要只罗列功能——要分析竞品为什么做出这些选择
- 定价情报需注明定价是公开的、按用量收费的，还是需联系销售获取
- 建议每季度更新一次分析——竞争格局变化很快
