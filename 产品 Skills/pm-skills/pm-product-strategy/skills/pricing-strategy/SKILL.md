## 定价策略（Pricing Strategy）

从价值交付、竞争定位和支付意愿出发，设计有依据的定价策略。

### 背景

你正在为 **$ARGUMENTS** 制定定价策略。

如果用户提供了文件（竞品定价、调研数据、财务模型或用量数据），请先阅读。如有需要，使用网络搜索调研竞品定价。

### 操作步骤

1. **理解交付的价值**：
   - 核心价值主张是什么？
   - 客户的替代方案是什么（以及其成本）？
   - 产品交付了哪些可量化的成果？（节省时间、创造收入、降低成本）
   - 基于该价值，客户的支付意愿是多少？

2. **评估定价模型**——推荐最适合的模型：

   | 模型 | 最适合 | 示例 |
   |---|---|---|
   | **统一费率（Flat-rate）** | 简单产品，成本可预期 | Basecamp（每月 $99 统一收费） |
   | **按席位（Per-seat）** | 协作工具、团队产品 | Slack、Figma |
   | **按用量（Usage-based）** | 基础设施、API 产品 | AWS、Twilio |
   | **分级定价（Tiered）** | 有明显用户细分的产品 | 多数 SaaS（免费/专业/企业） |
   | **免费增值（Freemium）** | 有病毒传播/网络效应的产品 | Spotify、Notion |
   | **免费增值 + 用量（Freemium + usage）** | 平台类产品 | Vercel、OpenAI API |
   | **价值定价（Value-based）** | 高影响力企业工具 | Salesforce、Palantir |

3. **分析竞争定价**：
   - 梳理竞品定价层级及各层包含内容
   - 确定你的产品定位（高端、中端还是性价比）
   - 发现定价空白或机会点
   - 注意行业定价惯例

4. **设计定价结构**：
   - **套餐分级**：定义 2-4 个层级，确保各层差异清晰
   - **功能分配**：哪些功能放在哪个套餐？（使用价值指标，而非任意限制）
   - **计费单元（Value Metric）**：按什么计费？（用户数、事件数、存储量、API 调用量）
   - **锚定定价**：让最受欢迎的套餐看起来是显而易见的最优选择
   - **年付折扣**：通常为月付价格的 15-20% 折扣

5. **估算价格敏感度**：
   - 范·韦斯滕多普价格敏感度计量仪（Van Westendorp PSM，如有调研数据）：
     - 太便宜 → 引发质量顾虑
     - 偏便宜 → 性价比高
     - 偏贵 → 开始犹豫
     - 太贵 → 不会购买
   - 若无数据，则基于竞品定价和交付价值进行估算

6. **规划定价实验**：
   - A/B 测试定价页面（不同价格点、套餐名称、功能组合）
   - 创始人主导的销售对话，测试支付意愿
   - 落地页测试不同价格锚点
   - 按价格点分析各组群的转化率

7. **输出定价建议**：
   ```
   推荐模型：[模型类型]
   计费单元：[收费依据]

   | 套餐 | 价格 | 目标细分 | 核心功能 | 定位 |
   |---|---|---|---|---|

   关键假设：
   - [假设] → [验证方法]

   风险：
   - [风险] → [缓解措施]
   ```

逐步思考。保存为 markdown 格式。标注上线前需要验证的假设。
