# /business-model -- 商业模式探索

使用四种互补框架构建并分析商业模式。可选择其中一种，也可全部运行以获得完整视图。

## 调用方式

```
/business-model lean 连接自由职业产品经理与初创公司的平台
/business-model full 企业级数据分析平台
/business-model startup 面向非英语母语用户的 AI 写作工具
/business-model value-prop SaaS 用户引导工具
/business-model all SaaS 用户引导工具        # 运行全部四种框架
/business-model                               # 询问你的需求
```

## 模式说明

---

### 精益画布模式（Lean Canvas Mode）

适用场景：早期阶段创意、初创公司、新产品线。

调用 **lean-canvas** 技能，生成完整的精益画布：

```
## 精益画布：[产品名称]

| 问题（Top 3） | 解决方案 | 独特价值主张（UVP） |
|--------------|---------|-------------------|
| 1. [问题]    | [对应解决方案] | [清晰简洁的核心信息] |
| 2. [问题]    |         |                   |
| 3. [问题]    |         |                   |

| 关键指标 | 不公平优势 |
|---------|---------|
| [你衡量的内容] | [竞争对手难以复制的部分] |

| 渠道 | 客户细分 |
|-----|---------|
| [如何触达客户] | [目标用户，优先列出早期采用者] |

| 成本结构 | 收入来源 |
|---------|---------|
| [固定成本 + 可变成本] | [如何赚钱] |

### 最高风险假设
[哪些条件必须成立才能让这个模式跑通——按风险优先级排序]

### 待运行的验证实验
[如何以低成本验证最高风险的假设]
```

### 完整商业模式画布模式（Full Business Model Canvas Mode）

适用场景：成熟产品、战略规划、投资材料。

调用 **business-model** 技能，生成全部 9 个模块：

```
## 商业模式画布（BMC）：[产品名称]

| 核心伙伴 | 核心活动 | 价值主张 | 客户关系 | 客户细分 |
|---------|---------|---------|---------|---------|
| [谁在帮助你] | [核心业务动作] | [客户为何选择你] | [如何与客户互动] | [你服务的对象] |

| 核心资源 | | 渠道 | |
|---------|---|-----|---|
| [你需要的资源] | | [如何交付价值] | |

| 成本结构 | 收入来源 |
|---------|---------|
| [你的成本] | [你的收入] |

### 分析
[该商业模式的优势与弱点]
[各模块如何相互强化]
[脆弱性与依赖关系]
```

### 创业画布模式（Startup Canvas Mode）

适用场景：需要在同一文档中兼顾策略与商业模式的新产品和初创公司。推荐优先于精益画布和 BMC 使用。

调用 **startup-canvas** 技能，生成包含 9 个策略模块 + 商业模式的创业画布：

```
## 创业画布：[产品名称]

### 第一部分：产品策略

| 愿景 | 市场细分 | 相对成本 |
|-----|---------|---------|
| [激励人心的为什么] | [JTBD，从第一个细分市场开始] | [低成本 vs 独特价值] |

| 价值主张 | 战略取舍 | 关键指标 |
|---------|---------|---------|
| [现状 → 解决方案 → 改变后 → 替代方案] | [你不做什么] | [北极星指标 + OMTM] |

| 增长 | 能力 | Can't/Won't（防御性测试） |
|-----|-----|------------------------|
| [产品驱动 vs 销售驱动，渠道] | [自建 vs 合作] | [竞争对手为何无法复制] |

### 第二部分：商业模式

| 成本结构 | 收入来源 |
|---------|---------|
| [固定 + 可变成本，以及如何规模化] | [定价模型，各渠道收入] |

### 策略一致性检验
[所有要素是否相互强化？]

### 最高风险假设
[哪些条件必须成立——以及如何验证]
```

---

### 价值主张模式（Value Proposition Mode）

适用场景：打磨产品信息、理解用户价值、产品-市场契合度分析。

调用 **value-proposition** 技能，生成基于 JTBD 框架的价值主张：

```
## 价值主张：[产品名称]

### 针对 [客户细分]：
1. **目标用户（Who）**：[目标用户画像]
2. **动机（Why）**：[他们正在尝试完成的任务]
3. **现状（What Before）**：[当前痛苦的现实处境]
4. **解决方案（How）**：[你的解决思路]
5. **改变后（What After）**：[改善后的处境]
6. **替代方案（Alternatives）**：[如果没有你他们会用什么，以及你的优势所在]

### 价值主张陈述
[一句话：为了 [目标用户]，他们有 [需求]，[产品] 是一款 [品类] 产品，能够 [核心价值]。不同于 [替代方案]，我们 [差异化优势]。]
```

---

### 全模式（All Mode）

运行全部四种框架，并增加一个综合分析模块，对比各框架的洞察异同。

## 工作流程（所有模式通用）

### 第一步：收集上下文

询问：
- 产品或业务创意是什么？
- 当前处于哪个阶段？（构思期、已验证、规模化中）
- 是否有现有商业模式需要优化？
- 目标客户是谁？

### 第二步：生成所选框架

按上述说明调用相关技能。

### 第三步：保存并迭代

保存为 markdown 格式。可继续提问：
- "帮我用 SWOT 或 PESTLE 分析来**压力测试这个商业模式**？"
- "帮我为收入来源**设计定价策略**？"
- "帮我围绕这个商业模式**构建策略画布**？"
- "帮我**识别桥头堡细分市场**？"

## 备注

- **创业画布** 是新产品的推荐起点——它将策略与商业模式分层，弥补了 BMC 和精益画布遗漏的内容（愿景、取舍、指标、Can't/Won't 防御性测试）
- **精益画布** 适合追求速度和假设验证——不必过度思考，但要注意它将策略和商业模式混在一个文档中
- **BMC** 更适合需要清晰呈现运营全貌的成熟业务，但缺少战略性模块（愿景、取舍、指标）
- **价值主张** 是产品-市场契合度对话中最锋利的工具
- 在"全模式"下，重点标注各框架达成共识的地方（强信号）以及出现分歧的地方（需深入研究）
