# /value-proposition -- 价值主张设计

使用 6 步 JTBD（用户待办任务）模板，为产品或功能设计清晰、有说服力的价值主张。这是 Strategyzer 价值主张画布的替代方案，以客户为起点，聚焦可落地的实际成果。

## 调用方式

```
/value-proposition 面向非英语母语用户的 AI 写作工具
/value-proposition [上传融资路演、PRD 或竞品分析]
/value-proposition                    # 直接询问你的产品情况
```

## 工作流程

### 第一步：了解产品和市场

接受以下形式的上下文输入：
- 产品描述（口头或文字）
- 上传文档（融资路演、PRD、竞品分析）
- 需要优化的现有价值主张

关键问题：
- 产品是做什么的？面向谁？
- 目前有哪些替代方案或变通做法？
- 你掌握哪些客户洞察或调研数据？

### 第二步：构建价值主张

调用 **value-proposition** 技能，生成 6 步模板：

```
## 价值主张：[产品名称]

### 针对 [客户细分]：

1. **目标用户（Who）**：[目标用户画像及特征]
2. **动机（Why）**：[他们正在尝试完成的任务，期望达成的结果]
3. **现状（What Before）**：[当前痛苦的处境——现有工具、摩擦点、变通做法]
4. **解决方案（How）**：[产品如何解决问题——交付价值的具体功能和能力]
5. **改变后（What After）**：[改善后的结果——有哪些新的可能]
6. **替代方案（Alternatives）**：[如果没有你，他们会用什么，以及你的优势所在]

### 价值主张陈述
[一句话：为了 [目标用户]，他们有 [需求]，[产品] 是一款 [品类] 产品，能够 [核心价值]。不同于 [替代方案]，我们 [差异化优势]。]

### 可复用的价值主张表述
- 营销端：[...]
- 销售端：[...]
- 用户引导端：[...]
```

如果用户有多个细分市场，为每个细分市场分别创建价值主张。

### 第三步：保存并提供后续建议

保存为 markdown 格式。可继续提问：
- "要我用**价值曲线**来**对比竞品**吗？"
- "要我围绕这个价值主张**构建完整的产品策略**吗？"
- "要我用这个**创建精益画布**或**创业画布**吗？"
- "要我从这些价值主张中**生成营销文案**吗？"

## 备注

- 本模板以客户为起点（目标用户/动机），逐步引导至解决方案——与 Strategyzer 画布将产品置于左侧的逻辑相反
- 每个价值主张都针对特定细分市场——不同细分市场有不同的价值主张
- 使用**价值曲线**（蓝海战略）可视化对比你的产品与竞品在各关键维度上的差异
- 价值主张是产品策略的一个组成部分——使用 `/strategy` 查看完整全貌
