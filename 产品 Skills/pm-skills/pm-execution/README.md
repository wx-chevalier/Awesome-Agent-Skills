# pm-execution

执行与产品管理技能：PRD（产品需求文档）、OKR（目标与关键成果）、路线图、Sprint（冲刺）、事前剖析（Pre-mortem）、干系人地图、用户故事、优先级框架等。

## Skills（技能，共 15 项）

- **brainstorm-okrs** — 围绕公司目标，集思广益制定团队级 OKR（目标与关键成果）。
- **create-prd** — 使用涵盖摘要、背景、目标、市场细分、价值主张、解决方案详情和发布计划的 8 节综合模板，创建产品需求文档（PRD）。
- **dummy-dataset** — 生成用于测试的逼真虚拟数据集，支持自定义列、约束条件及输出格式（CSV、JSON、SQL、Python 脚本）。
- **job-stories** — 使用"当 [情境] 时，我想要 [动机]，以便 [结果]"格式创建工作故事，并附带详细验收标准。
- **outcome-roadmap** — 将以输出为中心的路线图转化为以成果为中心的路线图。
- **pre-mortem** — 对 PRD（产品需求文档）进行事前剖析（Pre-mortem）（Pre-mortem）。
- **prioritization-frameworks** — 包含 9 种优先级框架的参考指南，提供公式、使用时机说明及模板。
- **release-notes** — 根据工单、PRD 或变更日志生成面向用户的发布说明。
- **retro** — 组织结构化的 Sprint（冲刺）回顾会。
- **sprint-plan** — 规划 Sprint（冲刺），包含容量估算、故事选取、依赖关系映射和风险识别。
- **stakeholder-map** — 使用权力/利益方格构建干系人地图，明确各象限的沟通策略，并生成沟通计划。
- **summarize-meeting** — 将会议记录整理为结构化模板，包含日期、参与者、主题、要点摘要和行动项。
- **test-scenarios** — 根据用户故事创建综合测试场景，涵盖测试目标、初始条件、用户角色、分步操作和预期结果。
- **user-stories** — 遵循 3C 原则（Card 卡片、Conversation 对话、Confirmation 确认）和 INVEST 标准创建用户故事，包含描述、设计链接和验收标准。
- **wwas** — 以"为什么-做什么-验收标准"（Why-What-Acceptance）格式创建产品待办事项。

## Commands（命令，共 10 条）

- `/pm-execution:generate-data` — 生成用于测试的逼真虚拟数据集，支持 CSV、JSON、SQL 插入语句或 Python 脚本格式。
- `/pm-execution:meeting-notes` — 将会议记录整理为结构化会议纪要，包含决策、行动项和后续跟进事项。
- `/pm-execution:plan-okrs` — 围绕公司目标，集思广益制定团队级 OKR（目标与关键成果）——定性目标与可量化关键成果。
- `/pm-execution:pre-mortem` — 对 PRD、发布计划或功能进行事前剖析（Pre-mortem）——在问题发生前识别潜在风险。
- `/pm-execution:sprint` — Sprint（冲刺）全生命周期管理——规划 Sprint、组织回顾会，或生成发布说明。
- `/pm-execution:stakeholder-map` — 在权力×利益方格上映射干系人，并制定定制化沟通计划。
- `/pm-execution:test-scenarios` — 根据用户故事或功能规格生成综合测试场景，涵盖正常流程、边界情况和错误处理。
- `/pm-execution:transform-roadmap` — 将以功能为中心的路线图转化为以成果为中心的路线图，以传达战略意图。
- `/pm-execution:write-prd` — 根据功能想法或问题陈述，创建全面的产品需求文档（PRD）。
- `/pm-execution:write-stories` — 将功能拆解为待办事项——用户故事、工作故事或 WWA 格式，附带验收标准。
