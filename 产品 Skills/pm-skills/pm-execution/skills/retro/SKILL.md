## Sprint 回顾会引导工具

运行结构化的回顾会，浮现洞察并产出可落地的改进措施。

### Context（背景）

你正在为 **$ARGUMENTS** 组织回顾会。

如果用户提供了文件（Sprint 数据、速率图表、团队反馈或历次回顾笔记），请先阅读。

### Instructions（操作指南）

1. **根据背景选择回顾格式**（或让用户选择）：

   **格式 A — Start / Stop / Continue（开始/停止/继续）**：
   - **开始**：我们应该开始做什么？
   - **停止**：我们应该停止做什么？
   - **继续**：哪些做得好，应该继续保持？

   **格式 B — 4Ls（喜欢/学到/缺少/期望）**：
   - **Liked 喜欢**：团队喜欢什么？
   - **Learned 学到**：获得了哪些新知识？
   - **Lacked 缺少**：缺少了什么？
   - **Longed For 期望**：希望拥有什么？

   **格式 C — 帆船模型**：
   - **风（推进力）**：什么在驱动我们前进？
   - **锚（阻力）**：什么在拖慢我们？
   - **礁石（风险）**：前方有哪些危险？
   - **岛屿（目标）**：我们想要到达哪里？

2. **如果用户提供了原始反馈**（如便利贴、问卷回复、Slack 消息）：
   - 将相似条目归类为主题
   - 识别被提及最多的话题
   - 注意情绪规律（挫败感、干劲、困惑）

3. **分析 Sprint 表现**：
   - Sprint 目标：是否达成？
   - 速率与承诺对比（过度承诺？承诺不足？）
   - 遇到的阻碍及解决方式
   - 协作模式（哪些有效，哪些无效）

4. **生成优先级行动项**：

   | 优先级 | 行动项 | 负责人 | 截止日期 | 成功指标 |
   |--------|-------|-------|---------|---------|
   | 1 | [具体、可落地的改进] | [姓名/角色] | [日期] | [如何判断是否有效] |

   - 限制在 2-3 个行动项（更多会没人执行）
   - 每个必须具体、可分配、可衡量
   - 如有历次回顾笔记，回顾上次行动项——是否已完成？

5. **生成回顾会摘要**：
   ```
   ## Sprint [X] 回顾会 — [日期]

   ### Sprint 表现
   - 目标：[达成 / 部分达成 / 未达成]
   - 承诺：[X 点] | 完成：[Y 点]

   ### 关键主题
   1. [主题] — [摘要]

   ### 行动项
   1. [行动] — [负责人] — [截止日期]

   ### 上次回顾跟进
   - [上次行动] — [状态：已完成 / 进行中 / 未开始]
   ```

保存为 Markdown 格式。保持建设性语气——目标是改进，而非追责。
