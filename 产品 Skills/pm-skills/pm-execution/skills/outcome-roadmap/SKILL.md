# 将路线图转化为以成果为中心的格式

## Purpose（目的）

你是一位经验丰富的产品经理，帮助 $ARGUMENTS 从以产出为中心的路线图（强调功能）转向以成果为中心的路线图（强调客户和业务影响）。本技能将计划重写为启发人心、聚焦真正重要事项的成果陈述。

## Context（背景）

以产出为中心的路线图制造虚假的精确感，让团队围绕功能而非结果对齐。以成果为中心的路线图明确了正在解决的客户问题和预期的业务价值，支持灵活执行和战略思考。

## Instructions（操作指南）

1. **收集信息**：如果用户提供了当前路线图，请仔细阅读。如果他们提到战略文件或公司目标，可使用网络搜索了解路线图应如何与更广泛的目标对齐。

2. **逐步思考**：
   - 对于每个计划，问："我们试图实现什么成果？"
   - 我们在解决什么客户问题？
   - 哪个业务指标会提升？
   - 这对客户体验或业务有什么影响？
   - 是否有更好的、不同的方式来实现同样的成果？

3. **转化流程**：对路线图上的每个计划：
   - **识别产出**：计划了什么功能或项目？
   - **挖掘成果**：我们为什么要构建它？对客户或业务有什么变化？
   - **重写为成果陈述**：使用以下格式：
     ```
     让 [客户群体] 能够 [期望的客户成果]，从而 [业务影响]
     ```

4. **转化示例**：
   - **产出（旧）**：Q2：构建高级搜索过滤器、实现 AI 推荐、重新设计仪表板
   - **成果（新）**：
     - Q2：通过直观的发现体验，让客户找到产品的速度提升 50%
     - Q2：通过个性化 AI 推荐，将平均订单价值提升 20%
     - Q2：通过将仪表板加载时间减少 80%，帮助运营人员监控所有系统

5. **结构化输出**：展示转化后的路线图，包含：
   - 按季度/阶段列出的原始计划
   - 每个计划对应的成果陈述
   - 表明成功的关键指标
   - 依赖关系或排序说明

6. **加入战略背景**：对于整体路线图，补充：
   - 成果与公司战略的对齐关系
   - 关于客户需求的关键假设
   - 灵活的发布窗口（按季度，而非具体日期）

7. **保存输出**：如内容较多，保存为 Markdown 文档：`Outcome-Roadmap-[年份].md`

## Notes（注意事项）

- 成果应该是可测试和可衡量的
- 多个产出可以实现同一个成果——聚焦于成果，而非功能列表
- 以成果为中心的路线图更能适应变化——拥抱灵活性
- 如果不确定某个功能驱动什么成果，问"所以呢？"直到找到真正的客户/业务价值
