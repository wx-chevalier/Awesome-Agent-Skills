# /interview -- 用户访谈准备与总结

两种模式：**prep（准备）** 在与用户交谈前创建结构化访谈脚本，**summarize（总结）** 在访谈结束后提炼关键洞察。

## 调用方式

```
/interview prep 企业用户的新手引导体验
/interview summarize [粘贴记录或上传文件]
/interview                    # 询问你需要哪种模式
```

## 模式说明

---

### 准备模式（Prep）

根据你的研究问题创建一份结构化访谈脚本。

#### 工作流程

**第一步：理解研究目标**

询问用户：
- 你希望了解什么？（具体的研究问题）
- 访谈对象是谁？（细分用户、职位角色、与产品的关系）
- 你有多少时间？（15 分钟、30 分钟、60 分钟）
- 本次研究将支持哪些决策？

**第二步：生成访谈脚本**

应用 **interview-script** 技能：

- 遵循"老妈测试"（The Mom Test）原则——询问他们的生活，而非评价你的想法
- 不引导性提问、不推销产品，聚焦过去的行为和真实情境
- 将脚本组织为以下几个部分：

```
## 访谈脚本：[研究主题]

**研究问题**：[我们希望了解的内容]
**访谈对象**：[受访者描述]
**时长**：[X] 分钟

### 破冰阶段（3-5 分钟）
[建立信任的问题、角色与背景了解]

### 核心探索（15-40 分钟）
[JTBD（用户待办任务）探测、过去行为、当前工作流程、痛点]
- 每个问题：问题本身 + 提问原因 + 追问提示

### 专项话题（5-10 分钟）
[针对特定功能或概念的定向问题——如有需要]

### 收尾阶段（3-5 分钟）
[开放式结束语、转介绍邀请、后续步骤]

### 记录模板
[用于在访谈中实时记录洞察的预格式化模板]

### 需警惕的红旗信号
[对话偏离轨道或受访者出于礼貌而非诚实作答的信号]
```

**第三步：定制与审阅**

- 根据时间调整问题数量
- 针对用户希望验证的具体假设添加追问
- 标注可能带有引导性的问题
- 提供可打印版本（保存为 markdown 文件到工作区）

---

### 总结模式（Summarize）

将访谈记录转化为结构化、可落地的洞察。

#### 工作流程

**第一步：接收访谈记录**

接受任意格式：
- **粘贴文本**：原始记录或访谈笔记
- **上传文件**：文档、文本文件或会议记录导出
- **口述摘要**：若用户描述了访谈内容（非完整记录）

如果输入是粗略笔记而非完整记录，在现有内容基础上工作，并说明局限性。

**第二步：提取与结构化**

应用 **summarize-interview** 技能：

解析记录，识别以下内容：
- **受访者画像**：职位角色、经验水平、用户细分、使用背景
- **用户待办任务（JTBD）**：受访者试图完成什么任务
- **当前工作流程**：他们现在如何解决问题
- **痛点**：挫折感、变通方法、时间黑洞
- **满意度信号**：哪些地方做得好、让人愉悦的时刻
- **原话引用**：捕捉关键洞察的逐字引用（如有时间戳更佳）
- **意外发现**：出乎意料或与假设相悖的内容
- **功能反应**：若讨论了具体功能或概念，记录受访者的反应

**第三步：生成访谈摘要**

```
## 访谈摘要

**受访者**：[匿名画像——职位、细分用户、经验]
**日期**：[如已知]
**时长**：[如已知]
**访谈者**：[如已知]

### 核心洞察
1. **[洞察]** — [支撑证据/引用]
2. **[洞察]** — [支撑证据/引用]
3. ...

### 用户待办任务（JTBD）
- **主要 JTBD**：[当我……时，我想要……，这样我就能……]
- **相关 JTBD**：[其他工作任务]

### 当前工作流程
[受访者当前如何逐步解决问题]

### 痛点
| 痛点 | 严重程度 | 引用 |
|------|----------|------|

### 满意度信号
| 有效之处 | 原因 | 引用 |
|----------|------|------|

### 值得关注的引用
> "[引用内容]" — 关于 [主题]

### 假设验证情况
| 假设 | 状态 | 证据 |
|------|------|------|

### 行动项
- [ ] [本次访谈的后续行动]
- [ ] [需进一步探索的研究问题]

### 原始笔记
[如有必要，包含带注释的关键段落]
```

将摘要保存为 markdown 文件。

**第四步：与更广泛的研究关联**

提供选项：
- "需要我**与你做过的其他访谈摘要进行对比**吗？"
- "要不要根据这位受访者的反馈**更新假设**？"
- "想要从多次访谈中**提炼用户画像**吗？"

## 注意事项

- 在准备模式中，始终包含"提问原因"注释——帮助访谈者保持方向感
- 在总结模式中，区分受访者"说了什么"与"做了什么"（行为数据优先于陈述数据）
- 标注同一访谈中的矛盾之处（说一套，做另一套）
- 如果记录中提到了竞品，记录相关竞争情报
- 在总结模式中，如果提供了多份记录，跨受访者总结，提炼共性规律
