# /brainstorm -- 多视角创意构思

从三个视角（产品经理、设计师、工程师）生成富有创意的产品创意或实验方案，根据你的产品是现有产品还是新产品进行针对性调整。

## 调用方式

```
/brainstorm ideas existing 移动银行 App 用户活跃度
/brainstorm ideas new 面向忙碌父母的 AI 饮食规划助手
/brainstorm experiments existing 新手引导流程改版
/brainstorm experiments new 面向自由设计师的交易平台
/brainstorm                          # 交互模式——询问你的需求
```

## 工作流程

### 第一步：确定模式

解析参数，识别两个维度：

1. **头脑风暴内容**：`ideas`（功能创意）或 `experiments`（验证实验）
2. **产品阶段**：`existing`（持续探索）或 `new`（初始探索）

如果任一维度缺失，向用户询问。如果两者都缺失，提问：
- "你是在构思**创意**（要做什么），还是设计**实验**来验证假设？"
- "这是针对**现有产品**还是**新产品概念**？"

### 第二步：收集背景信息

向用户询问背景信息，保持对话感——优先问最关键的问题：

**现有产品：**
- 产品是什么？当前用户是谁？
- 你在探索哪个机会方向或问题空间？
- 有哪些约束条件？（技术债、平台限制、团队容量）
- 之前尝试过什么？

**新产品：**
- 产品概念是什么？解决什么问题？
- 目标用户是谁？他们现在用什么替代方案？
- 目前处于哪个阶段？（概念草图、已验证问题、早期原型）
- 最高风险的假设是什么？

接受上传文件（PRD（产品需求文档）、调研报告、战略文档）、粘贴文本或对话内容作为背景资料。

### 第三步：生成输出

**若构思创意** — 应用 **brainstorm-ideas-existing** 或 **brainstorm-ideas-new** 技能：
- 从三个视角生成创意：产品经理（用户价值、商业影响）、设计师（用户体验、愉悦感、无障碍设计）、工程师（技术创新、平台能力、可扩展性）
- 每个创意包含：名称、描述、用户影响评估、可行性评估
- 筛选并排列前 5 个创意，附理由说明
- 标注哪些是快速见效型，哪些是战略型长期投入

**若设计实验** — 应用 **brainstorm-experiments-existing** 或 **brainstorm-experiments-new** 技能：
- 现有产品：建议 A/B 测试、原型测试、假门测试、绿野仙踪测试、礼宾式实验和技术探针
- 新产品：创建 XYZ+S 假设并建议预型实验（落地页、说明视频、预购、礼宾式 MVP）
- 每个实验包含：假设陈述、方法、成功标准、工作量估算、预计周期
- 按学习收益/工作量比排序

### 第四步：深化与迭代

展示初步结果后，提供选项：
- "需要我把某个创意**详细展开**成完整规格吗？"
- "要不要**识别**头部创意背后的假设？"（链接到 `identify-assumptions-existing` 或 `identify-assumptions-new` 技能）
- "想要**设计实验**来验证头部创意吗？"（切换到实验模式）
- "要不要**与当前待办列表对比排优先级**？"（链接到 `prioritize-features` 技能）

## 输出格式

### 创意输出：
```
## 头脑风暴：[产品/功能方向]
**模式**：[现有/新产品]创意构思
**背景**：[1-2 句摘要]

### 产品经理视角
1. **[创意名称]** — [描述] | 影响力：[高/中/低] | 工作量：[高/中/低]
2. ...

### 设计师视角
1. **[创意名称]** — [描述] | 影响力：[高/中/低] | 工作量：[高/中/低]
2. ...

### 工程师视角
1. **[创意名称]** — [描述] | 影响力：[高/中/低] | 工作量：[高/中/低]
2. ...

### 前 5 名推荐
| 排名 | 创意 | 推荐理由 | 快速见效？ |
|------|------|----------|-----------|

### 后续步骤
[如何处理这些创意]
```

### 实验输出：
```
## 实验设计：[产品/功能方向]
**模式**：[现有/新产品]实验设计

### 假设陈述
1. **[假设]** — XYZ 格式：[X]% 的 [Y] 将在 [S 时间范围内] 做 [Z]

### 推荐实验
| # | 实验 | 验证的假设 | 方法 | 工作量 | 时间周期 |
|---|------|-----------|------|--------|----------|

### 实验详情
[每个实验：实验设置、成功标准、风险、预期收获]
```

## 注意事项

- 现有产品的创意应基于当前用户行为和已验证的问题
- 新产品应优先聚焦在可欲性（desirability）和技术可行性风险
- 如果用户上传了调研报告或访谈记录，先提取洞察再构思
- 先求广度，再求深度——先生成大量创意，再进行评估
