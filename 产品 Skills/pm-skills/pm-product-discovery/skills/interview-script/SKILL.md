## 用户访谈脚本

创建一份能挖掘真实洞察而非主观意见的结构化访谈脚本。遵循"老妈测试"（The Mom Test）原则——询问他们的生活，而非评价你的想法。

### 领域背景

用户访谈是持续探索**第一阶段（探索）**中的信息来源之一。其他来源包括：利益相关方访谈、使用数据分析、数据分析、用户调研、市场趋势、SEO/SEM 分析。产品经理需要直接接触用户、利益相关方、工程师和设计师——"不通过代理人"。**产品铁三角**（产品经理 + 设计师 + 工程师 — Teresa Torres）应共同参与探索工作，而不只是产品经理单独进行。

### 背景

你正在为 **$ARGUMENTS** 的调研准备用户访谈脚本。

如果用户提供了文件（用户画像、假设清单、产品简报或以往访谈笔记），请先阅读。

### 操作步骤

1. **澄清研究目标**：
   - 团队需要回答哪些具体问题？
   - 本次研究将支持哪些决策？
   - 哪些假设需要验证？

2. **创建包含以下部分的访谈脚本**：

   ### 开场白（2-3 分钟）
   - 介绍自己和访谈目的（学习，不是销售）
   - 设定预期："没有对错之分，我们只是希望从您的经历中学习。"
   - 征得录音许可（如适用）
   - 确认可用时间

   ### 破冰阶段：背景与角色（5 分钟）
   - "请介绍一下您的工作职责，以及典型的一天/一周是什么样的。"
   - "您从事[与产品领域相关的活动]多久了？"
   - 目标：建立信任，理解受访者的背景

   ### 核心探索：用户待办任务（JTBD）（15-20 分钟）

   **当前情境与行为**（过去时态，具体实例）：
   - "请带我回顾一下您上次[我们正在探索的事情]的经过。发生了什么？"
   - "您用了哪些工具或方法？"
   - "花了多长时间？有哪些人参与？"

   **痛点与挫败感**（观察，不引导）：
   - "那个过程中最难的地方是什么？"
   - "如果有一根魔法棒，您希望改变什么？"
   - "您曾尝试过哪些解决方法？结果如何？"

   **期望产出**（用他们自己的语言）：
   - "对您来说，这个领域'做得好'是什么样子？"
   - "您怎么判断这件事是否进展顺利？"

   **支付意愿 / 优先级**（真金白银的投入）：
   - "您目前在这上面花了多少时间/金钱？"
   - "您有没有寻找过更好的解决方案？找到了什么？"
   - "为了解决这个问题，您愿意放弃什么？"

   ### 追问技巧
   当挖到有价值的线索时使用：
   - **"能再说详细一点吗？"** — 可以展开任何话题
   - **"为什么？"**（温和地问，2-3 次）— 深挖根本原因
   - **"能举个具体例子吗？"** — 从观点转向事实
   - **"接下来发生了什么？"** — 顺着故事追问
   - **"这让您有什么感受？"** — 捕捉情绪强度

   ### 老妈测试原则
   - 询问**他们的生活**，而非你的想法
   - 询问**过去的事**，而非未来（"你会用 X 吗？"毫无价值）
   - **少说多听** — 目标是 80/20 分配
   - 访谈过程中**绝不推销**
   - 关注**强烈情绪** — 它们代表真实的痛点或愉悦感
   - **称赞是噪音** — "听起来很酷！"什么都说明不了

   ### 收尾阶段（3-5 分钟）
   - "有没有什么我没有问到、但您认为很重要的事情？"
   - "还有谁是我应该就这个话题去交流的？"
   - 感谢受访者的时间
   - 说明后续步骤（如有）

3. **定制脚本**：根据具体产品方向、用户画像和研究目标调整问题。根据可用的访谈时长增减部分。

4. **附上记录模板**：
   ```
   受访者：[姓名 / 编号]
   日期：[日期]
   核心工作任务：[他们试图完成什么]
   当前解决方案：[他们现在使用什么]
   最大痛点：[他们的第一大挫败]
   期望产出：[成功在他们看来是什么样的]
   支付意愿：[他们目前投入了多少/愿意投入多少]
   意外发现：[出乎意料的内容]
   后续跟进：[下一步行动]
   ```

保存为 markdown。脚本和记录模板均包含在内。
