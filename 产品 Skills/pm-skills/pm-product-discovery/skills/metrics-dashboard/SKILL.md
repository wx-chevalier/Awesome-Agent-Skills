## 产品指标看板

设计一套完整的产品指标看板，包含正确的指标选取、可视化方式和告警阈值。

### 背景

你正在为 **$ARGUMENTS** 设计指标看板。

如果用户提供了文件（现有看板、分析数据、OKR（目标与关键结果）或战略文档），请先阅读。

### 领域背景

**指标 vs KPI vs 北极星指标**：指标（Metrics）= 所有可衡量的事物。KPI（关键绩效指标）= 在较长时间维度上追踪的少数关键量化指标。北极星指标（North Star Metric）= 单一以用户为中心的 KPI，是业务成功的领先指标。

**好指标的 4 个标准**（Ben Yoskovitz，《Lean Analytics》）：(1) 易理解——形成共同语言；(2) 可比较——随时间变化，而非某个时间点的快照；(3) 比率或比例——比绝对数字更具洞察力；(4) 能改变行为——黄金法则："如果一个指标不会改变你的行为，它就是一个糟糕的指标。"

**8 种指标类型**：虚荣指标 vs 可行动指标（只有可行动指标才能改变行为）、定性 vs 定量（WHAT vs WHY——两者都需要；永远不要停止与客户交谈）、探索型 vs 报告型（探索数据以发现意外洞察）、滞后 vs 领先（领先指标能加快学习周期，例如客户投诉预示流失）。

**5 个行动步骤**：(1) 用好指标的 4 个标准审计现有指标；(2) 更新看板——确保所有关键指标都满足标准；(3) 识别虚荣指标——谨慎使用；(4) 区分领先与滞后指标；(5) 选定一个问题，深度挖掘数据。

案例研究和更多详情请参阅：Ben Yoskovitz 的 [《你追踪的指标对吗？》](https://www.productcompass.pm/p/are-you-tracking-the-right-metrics)

### 操作步骤

1. **识别指标框架** — 将指标分层组织：

   **北极星指标**：最能体现核心价值交付的单一指标

   **输入指标**（3-5 个）：驱动北极星指标的关键杠杆

   **健康指标**：确保整体产品健康的护栏指标

   **业务指标**：收入、成本和单位经济

2. **对每个指标进行定义**：

   | 指标 | 定义 | 数据来源 | 可视化类型 | 目标值 | 告警阈值 |
   |---|---|---|---|---|---|
   | [名称] | [精确计算方式：分子/分母、时间窗口] | [数据出处] | [折线图 / 柱状图 / 数字 / 漏斗图] | [目标值] | [触发告警的条件] |

3. **设计看板布局**：

   ```
   ┌─────────────────────────────────────────────┐
   │  北极星指标：[指标] — [当前值]               │
   │  趋势：[↑/↓ 较上期变化 X%]                  │
   ├──────────────────┬──────────────────────────┤
   │  输入指标 1      │  输入指标 2              │
   │  [迷你折线图]    │  [迷你折线图]            │
   ├──────────────────┼──────────────────────────┤
   │  输入指标 3      │  输入指标 4              │
   │  [迷你折线图]    │  [迷你折线图]            │
   ├──────────────────┴──────────────────────────┤
   │  健康：[延迟] [错误率] [NPS（净推荐值）]     │
   ├─────────────────────────────────────────────┤
   │  业务：[MRR（月经常性收入）] [CAC（获客成本）] [LTV（用户生命周期价值）] [流失率] │
   └─────────────────────────────────────────────┘
   ```

4. **设定评审节奏**：
   - **每日**：运营健康状况（错误、延迟、关键流程）
   - **每周**：输入指标和参与度趋势
   - **每月**：北极星指标、业务指标、OKR（目标与关键结果）进展
   - **每季度**：战略回顾和指标重新校准

5. **定义告警规则**：
   - 哪些阈值触发调查？
   - 谁来接收告警，通过什么渠道？
   - 预期的响应时间是多少？

6. **根据用户背景推荐工具**：
   - 产品分析：Amplitude、Mixpanel、PostHog
   - SQL 看板：Looker、Metabase、Mode
   - 运营健康：Datadog、Grafana

逐步思考。将看板规格文档保存为 markdown。
