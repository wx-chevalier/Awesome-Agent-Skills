## 头脑风暴产品创意（现有产品）

面向持续产品探索的多视角构思法，从产品经理、设计师和工程师三个视角生成创意，再从中筛选出最优的五个。

### 背景

你正在支持一个产品铁三角，对 **$ARGUMENTS** 进行持续产品探索。

如果用户提供了文件（调研数据、机会解决方案树、用户画像），请先阅读。如果提到了产品 URL，使用网络搜索了解该产品。

### 领域背景

**产品铁三角**（Teresa Torres，《Continuous Discovery Habits》）：产品经理 + 设计师 + 工程师共同参与探索。"最好的创意往往来自工程师。"探索并非线性——实验失败时要循环回去。使用**机会解决方案树（Opportunity Solution Tree）**（Teresa Torres）来映射机会 → 解决方案 → 实验。

### 操作步骤

用户将描述目标、目标细分市场和期望产出。请按以下步骤处理：

1. **理解机会点**：确认产品、目标、市场细分和期望产出。如有歧义，请求澄清。

2. **从三个视角构思** — 每个视角各生成 5 个创意：
   - **产品经理**：聚焦商业价值、战略契合度和客户影响
   - **产品设计师**：聚焦用户体验、可用性和愉悦感
   - **软件工程师**：聚焦技术可能性、数据利用和可扩展方案

3. **跨视角筛选前 5 个创意**，依据：
   - 与既定目标的战略契合度
   - 对期望产出的潜在影响
   - 可行性和所需工作量
   - 与现有解决方案的差异化程度

4. **针对每个优选创意**，提供：
   - 清晰的名称和一句话描述
   - 被选中的理由（推理过程）
   - 需要验证的关键假设

逐步思考。以清晰、结构化的格式呈现创意。

如果输出内容较多，保存为用户工作区中的 markdown 文档。
