# pm-product-discovery

面向产品经理的产品探索技能包：创意构思、实验设计、假设验证、功能优先级排序及用户访谈整理。

## 技能清单（13 个）

- **analyze-feature-requests** — 按主题、战略契合度、影响、工作量和风险对功能需求列表进行分析与优先级排序。
- **brainstorm-experiments-existing** — 为现有产品设计假设验证实验。
- **brainstorm-experiments-new** — 为新产品设计精益创业实验（预型实验）。
- **brainstorm-ideas-existing** — 通过多视角构思法（产品经理、设计师、工程师）为现有产品头脑风暴产品创意。
- **brainstorm-ideas-new** — 在初始探索阶段为新产品头脑风暴功能创意。
- **identify-assumptions-existing** — 从价值（Value）、可用性（Usability）、可行性（Viability）和技术可行性（Feasibility）四个维度识别现有产品功能创意中的高风险假设。
- **identify-assumptions-new** — 跨越 8 个风险类别（含上市策略、战略规划和团队能力）识别新产品创意中的高风险假设。
- **interview-script** — 创建结构化的用户访谈脚本，包含 JTBD（用户待办任务）探测问题、破冰环节、核心探索及收尾部分。
- **metrics-dashboard** — 定义并设计产品指标看板，包含核心指标、数据来源、可视化类型和告警阈值。
- **opportunity-solution-tree** — 构建机会解决方案树（OST），将期望产出映射到机会、解决方案和实验，以结构化产品探索工作。
- **prioritize-assumptions** — 使用影响力 × 风险矩阵对假设进行优先级排序，并为每项假设推荐实验方案。
- **prioritize-features** — 基于影响力、工作量、风险和战略契合度对功能创意列表进行优先级排序。
- **summarize-interview** — 将用户访谈记录整理为结构化模板，包含 JTBD（用户待办任务）、满意度信号和行动项。

## 命令清单（5 个）

- `/pm-product-discovery:brainstorm` — 从产品经理、设计师和工程师视角头脑风暴产品创意或实验方案，适用于现有产品或新产品。
- `/pm-product-discovery:discover` — 运行完整的产品探索流程——从创意构思到假设映射再到实验设计。
- `/pm-product-discovery:interview` — 准备用户访谈脚本，或将访谈记录整理为结构化洞察。
- `/pm-product-discovery:setup-metrics` — 设计产品指标看板，包含北极星指标、输入指标、健康指标和告警阈值。
- `/pm-product-discovery:triage-requests` — 分析、归类并对来自客户或利益相关方的功能需求进行批量优先级排序。
