# /review-resume -- 产品经理简历评审

基于产品管理岗位最佳实践对简历进行深度评审。评估结构、量化成果、关键词优化，并提供具体的改进建议与示例。

## 调用方式

```
/review-resume [粘贴简历文本]
/review-resume [上传简历 PDF 或 DOCX]
```

## 工作流程

### 第一步：接收简历

接受粘贴文本、上传的 PDF 或 DOCX 文件，解析完整内容。

### 第二步：对照 10 项最佳实践评估

应用 **review-resume** 技能：

1. **量化成果**：成就是否有数据支撑？（营收、用户数、转化率等）
2. **XYZ+S 公式**："通过 [Z 做法] 利用 [S 技能/工具] 实现了 [X 成果]，体现在 [Y 指标] 上"
3. **产品经理专业表述**：使用产品术语（已上线、主导用户调研、制定策略）
4. **结构与可读性**：板块清晰、格式统一、便于快速浏览
5. **关键词优化**：与产品经理岗位招聘描述中的常见关键词匹配
6. **职业叙事**：体现职业发展脉络和不断扩大的影响范围
7. **简洁性**：应届/初级一页，资深不超过两页，不堆砌废话
8. **岗位相关性**：经历聚焦产品方向，而非泛泛的职责描述
9. **技术可信度**：体现与研发、数据、设计团队的协作经历
10. **领导力信号**：跨职能影响力、干系人管理、人才培养

### 第三步：生成评审报告

```
## 简历评审

**综合评分**：[X/10]
**最强项**：[对应最佳实践]
**最大提升空间**：[对应最佳实践]

### 评分卡
| # | 最佳实践 | 得分 | 评估说明 |
|---|---------|------|---------|
| 1 | 量化成果 | [/10] | [简要评估] |
| 2 | XYZ+S 公式 | [/10] | [简要评估] |
| ... | ... | ... | ... |

### 重点改进建议（前 3 条）

**1. [影响最大的改动]**
- 原文：「[简历中的原始文本]」
- 建议改为：「[优化后的版本]」
- 理由：[说明原因]

**2. [第二项改进]**
[同上格式]

**3. [第三项改进]**
[同上格式]

### 逐板块反馈
[针对简历各板块的具体意见：个人简介、工作经历、教育背景、技能]

### 缺失内容
[产品经理简历应有但目前缺少的内容]

### 建议补充的关键词
[简历中缺少但在典型招聘描述中频繁出现的产品经理相关关键词]
```

### 第四步：提供后续选项

- "是否需要我**针对某个特定职位描述对这份简历进行定向优化**？"
- "是否需要我**用 XYZ+S 公式改写具体的经历条目**？"
- "是否需要我**根据这份简历生成一封求职信**？"

## 注意事项

- 反馈要具体且有建设性——"加数据"没有意义，"将'优化了新手引导'改为'将新手引导流失率降低 23%（从每月 450 人降至 347 人）'"才是可执行的建议
- 产品经理简历应强调结果而非产出，强调影响力而非职权
- ATS（简历筛选系统）优化很重要——关键词要自然融入行文
- 不同级别的产品经理有不同侧重：APM（助理产品经理）看潜力，高级产品经理看影响力，总监及以上看规模与战略
