# /tailor-resume -- 简历与职位描述匹配优化

读取您的简历和目标职位描述，策略性地对齐您的经历，最大化获得面试机会。包含关键词优化、经历条目改写和差距分析。

## 调用方式

```
/tailor-resume [上传简历] 职位描述如下：[粘贴职位描述]
/tailor-resume [同时上传简历和职位描述文件]
```

## 工作流程

### 第一步：接收两份文档

需要两份输入：
- 简历（文本、PDF 或 DOCX）
- 目标职位描述（文本、URL 或文件）

如只提供了其中一份，请求另一份。

### 第二步：解析职位描述

提取以下信息：
- 必要资质与技能
- 优先资质
- 核心职责
- 行业与业务领域信号
- 职级指标
- 团队文化与协作风格信号

### 第三步：优化简历

应用 **review-resume** 技能：

- **关键词对齐**：将职位描述中的关键词映射到简历内容，自然地补充缺失的关键词
- **经历条目改写**：用 XYZ+S 公式重新表述经历，突出与职位相关的成就
- **板块排序调整**：将最相关的经历优先呈现
- **个人简介/求职意向**：改写为直接回应目标岗位
- **技能板块**：与职位要求对齐

### 第四步：生成优化简历及分析报告

```
## 简历定向优化：[职位名称] @ [公司名称]

### 匹配度评分：[X/10]

### 关键词差距分析
| 职位描述关键词 | 简历中是否包含 | 建议 |
|--------------|--------------|------|

### 改动说明
1. **[板块]**：[改了什么以及改动原因]
2. ...

### 优化后的简历
[完整改写后的简历文本]

### 差距分析
**强匹配项**：[您的经历与职位要求直接吻合的部分]
**重新定位项**：[经历经过调整后与职位要求契合的部分]
**差距项**：[职位要求中您没有明确体现的内容——附改进建议]

### 求职信写作要点
[3-4 个在求职信中需要重点强调、用于弥补剩余差距的要点]
```

将优化后的简历保存为 Markdown 格式。

## 注意事项

- 绝不捏造经历——真实调整表述方式，不虚构内容
- 个人简介/求职意向是每次投递中投入产出比最高的板块，值得重点打磨
- 尽量使用职位描述中的原始表述（对方写"跨职能协作"，你就用"跨职能协作"）
- 高级职位着重体现规模与战略影响力；执行层岗位着重体现亲手操盘的落地能力
