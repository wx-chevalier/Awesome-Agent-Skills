# /draft-nda -- NDA（保密协议）起草

根据您的具体情况起草专业的保密协议（NDA）。涵盖信息类型、管辖区、有效期，并明确标注需要法律审查的条款。

## 调用方式

```
/draft-nda 我们的初创公司与潜在企业客户之间签订的互相保密协议
/draft-nda 外包承包商访问我们代码库的单向保密协议
```

## 工作流程

### 第一步：收集背景信息

询问以下问题：
- 双方是谁？（公司名称及各方角色）
- 互相保密还是单向保密？
- 需要保护哪些信息？（商业秘密、代码、业务数据、客户数据）
- 适用管辖区？（适用的州/国家法律）
- 有效期限？（保密义务持续多长时间）
- 是否有特殊关切？（竞业禁止、禁止招揽、知识产权归属）

### 第二步：起草 NDA

应用 **draft-nda** 技能：

生成完整的 NDA，涵盖以下内容：
- 双方信息及序言
- 保密信息的定义（含具体示例）
- 接收方的保密义务
- 例外情形（公开信息、独立开发等）
- 有效期与存续条款
- 资料的归还/销毁
- 违约救济措施
- 适用法律与管辖区
- 标准通用条款（可分割性、完整协议、修改方式）

### 第三步：交付成果

```
## 保密协议（NDA）

[完整 NDA 正文，附标注说明]

### 需法律审查的条款
| 条款 | 需审查的原因 | 注意事项 |
|------|------------|---------|

### 通俗语言摘要
[用简单易懂的语言说明本 NDA 的含义，便于非法律专业人士理解]
```

保存为 Markdown 格式。可提供导出为 DOCX 格式以供签署。

## 注意事项

- 本文档仅为起点——始终建议请合格的法律顾问审查
- 涉及重大法律风险的条款用 ⚠️ 标记
- 加入通俗语言注释，让非法律专业人士也能理解所签内容
- 互相保密协议通常更受青睐——更公平，谈判也更顺畅
