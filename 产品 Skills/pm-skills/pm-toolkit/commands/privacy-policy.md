# /privacy-policy -- 隐私政策生成器

为您的产品起草完整的隐私政策。涵盖数据类型、管辖区、合规要求（GDPR、CCPA 等），并标注需法律审查的条款。

## 调用方式

```
/privacy-policy 面向美国和欧盟用户的 SaaS 分析工具，收集用户行为数据
/privacy-policy 集成第三方服务并采集位置数据的移动应用
```

## 工作流程

### 第一步：收集背景信息

询问以下问题：
- 产品或服务是什么？
- 收集哪些数据？（个人信息、使用数据、Cookie、位置信息、支付信息）
- 用户在哪里？（决定适用的法规：GDPR、CCPA 等）
- 是否涉及第三方数据共享？（分析服务、广告、系统集成）
- 数据如何存储？存储在哪里？保留多久？
- 是否有年龄限制？（COPPA 相关考量）

### 第二步：起草隐私政策

应用 **privacy-policy** 技能：

生成涵盖以下内容的各项条款：
- 收集哪些数据及收集方式
- 数据用途（目的说明）
- 处理数据的法律依据（GDPR）
- 数据共享与第三方
- 数据保留与删除
- 用户权利（访问、删除、可携性、退出）
- Cookie 政策
- 安全措施
- 儿童隐私（如适用）
- 跨境数据传输
- 联系方式
- 政策更新流程

### 第三步：交付成果

```
## 隐私政策：[产品名称]

[完整政策正文]

### 合规检查清单
| 法规 | 状态 | 备注 |
|------|------|------|

### 需法律审查的条款
| 条款 | 原因 | 优先级 |
|------|------|--------|

### 实施检查清单
- [ ] Cookie 同意横幅
- [ ] 数据主体请求处理流程
- [ ] 数据处理记录
- [ ] 与数据处理方签订 DPA（数据处理协议）
```

保存为 Markdown 格式。可提供 DOCX 导出。

## 注意事项

- 本文档为模板——发布前请务必经法律顾问审查
- GDPR 和 CCPA 有具体且不可近似处理的要求——需标注必须请专家审查的位置
- 隐私政策应使用通俗语言，而非法律术语
- 数据处理实践发生变化时即应更新政策，不要等到年度例行更新
