# /proofread -- 语法与行文检查

识别文本中的语法、逻辑和行文问题。提供具体、针对性的修改建议，不对整篇文档进行改写。

## 调用方式

```
/proofread [粘贴文本]
/proofread [上传文档]
```

## 工作流程

### 第一步：接收文本

接受任意形式的文本：直接粘贴、上传文档（DOCX、PDF、Markdown）或邮件草稿。

### 第二步：分析

应用 **grammar-check** 技能：

扫描三类问题：

**语法**：拼写、标点、主谓一致、时态一致性、助词用法
**逻辑**：前后矛盾、无依据的断言、循环论证、指代不清
**行文**：过渡生硬、句式节奏、段落结构、重复冗余、可读性

### 第三步：输出问题报告

```
## 校对报告

**文本长度**：[字数]
**发现问题**：[按类别统计数量]

### 问题列表

#### 1. [类别：语法/逻辑/行文]
- **位置**：「[含问题的原文引用]」
- **问题**：[具体说明]
- **修改建议**：「[修改后的文本]」

[每个问题依此格式重复]

### 汇总
- 语法：[X] 处
- 逻辑：[X] 处
- 行文：[X] 处
- 整体质量：[综合评估]
```

### 第四步：提供后续选项

- "是否需要我**应用所有修改**并返回修订后的完整文本？"
- "是否需要我**重点深入检查某个段落**？"

## 注意事项

- 修改建议应尽量精简——只改有问题的地方，保留作者的写作风格
- 对于非母语写作者，尤其要清楚说明*为何*建议修改
- 不要过度纠正风格偏好——"有所不同"和"有错误"是两回事
- 对于正式文档，还需检查语气一致性和受众适配性
